# 氛围编程？编毛咧！

这个标题是用来吸引眼球的！原标题应该是《从氛围编程到超级应用》。

实际上，我认为氛围编程还会再流行一阵子，甚至在某个时间点会成为软件开发的主流。\
去年2025年初时，AI的代码还差强人意，时不时在一些简单的地方掉链子。\
现在进入2026年，AI的进步巨大，很快就会把绝大部分开发者踢出局，也会成为众多软件公司的掘墓人。\
前些日子的SAAS股价大跌，预示着淘汰赛已经开始了。

然而，正如标题所要表达的，我认为氛围编程也只是一个过渡阶段的产物。

在软件开发里，我们曾多次重复这样的模式：
1. 起初，手工编写大量机械雷同的代码，质量完全取决于开发者的水平；
2. 然后，出现了好多高效的工具和成熟的库包，使得高质量的代码生成变得更容易；
3. 最后，随着技术的进步，这样的代码生成也渐渐消失了；

氛围编程的产物，那些应用，类比上文中的代码。在迎来一个全面开花的高潮后，我想也会渐渐的（迅速的）消失。

想想我们每时每刻使用的应用，\
要打开这个娱乐，使用那个购物，在某些个应用上工作，在其它几个社交，还有健身，理财 ... 氛围编程的流行，会带来越来越多雷同的应用 ... \
假如，有一个聪明能干的“管家”在身边，凡事你只需指示它去完成，甚至它可能在你想到前已经处理完毕，规划妥当了。\
那么，谁还要装这个，用那个，在块小玻璃上点来戳去的呢？！

在我的设想里，取而代之的以AI为核心驱动的超级应用。\
它具备以下特征：
### AI驱动
这是核心——由一个存在于服务对象本地的AI驱动。\
这个AI负责收集信息，查询知识库，使用工具，分析处理当前任务和未来规划。\
它会和其它的私有AI，公共AI，在网络间保持着沟通联系，并且互相学习，自动（自我？）进化升级 ...

### 静态知识
<a href="/creation/detail?id=gg-a-5" target="_blank">另篇文章</a>里聊过，静态的知识应该和推理系统分离的独立系统，是无须训练的，持续积累的，无论私有的还是世界的，全部供给推理系统分析使用的。


### 使用与制造工具
既然是AI，理所当然的应该有使用和制造工具的能力。\
就是说，不该每次都从头推理判断思考问题，而是和人类一样，选择使用已有的工具来解决。\
各种类似的可以实现这个目标的规范概念协议，最早是Function Call，后来A2A和MCP，然后Skill，最近又蹦出来Contract ... 实在很是需要一个通用的业界标准。

同样和人类一样，除了使用工具外，还需要发明创造和持续改进工具的能力。\
这里就类似上面的氛围编程，只是主角从人类变成AI。\
从这个意义上讲，应用不该是消失了，而是从人类使用的“工具”，变成了AI使用的“工具”了。
