# 氛围编程？编毛咧！

这个标题是用来吸引眼球的，^O^;
实际上，我认为氛围编程在接下来的几年里，会愈发的流行起来，甚至会成为软件开发的主流。

直到25年初时，AI的代码还差强人意，时不时卡在一些简单的地方让人哭笑不得。如今的AI进步巨大，方方面面已经超过了人类——至少是我自己的水平了。它很快就会把绝大部分开发者踢出局，最终成为众多软件公司的掘墓人。最近的SAAS股价大跌预示着淘汰赛已经开始了。

然而，正如标题所要表达的，我认为氛围编程也只是一个过渡阶段的产物。

在软件开发的历史上，我们曾重复这样的模式：
1. 手工编写大量机械重复的代码，质量完全取决于开发者的水平；
2. 出现了好多高效的工具/成熟的库包，使得高质量的代码生成变得更容易；
3. 最后，随着技术的进步，这些机械重复的代码生成也基本消失了；

氛围编程的产物，那些各式各样各行各业的应用，类比上文中的机械重复的代码。
1. AI出现之前，需要各种编程知识和技能来开发软件；
2. AI出现之后，氛围编程，这个工具让高质量的应用开发变得更容易；
3. 最后，随着AI能力的提升，这些应用——氛围编程的产物也会大多会消失了；

就说我们每时每刻在使用应用吧：
打开这个娱乐；使用那个购物；在某些个应用上工作；在其它几个社交；还有健身，理财……
假如，有一个聪明能干的“管家”（AI）在身边，凡事你只需指示它去完成，甚至它可能在你想到前已经处理完毕，规划妥当了。
那么，谁还要装这个，用那个，点来戳去的呢？！

从这个意义上讲，应用不该是消失了，而是从人类使用的“工具”，变成了AI使用的“工具”了。

取决于，AI算力的性价比

所以，氛围编程最终也会被取代，随之消失。
